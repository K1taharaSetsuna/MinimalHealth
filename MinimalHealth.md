# 产品设计文档 (PRD)：极简健康助手 Pro (MinimalHealth Pro)

## 1. 产品概述与设计哲学
*   **产品定位**：一款全面但视觉极简的个人健康、饮食与运动追踪管理 App。
*   **设计哲学**：
    *   **单页面单任务 (One Task Per Screen)**：拒绝信息过载，一个界面只引导用户做一件事。
    *   **大面积留白 (Negative Space)**：用留白代替边框，靠字号粗细区分视觉层级。
    *   **触觉补足 (Haptic Feedback)**：在极简视觉下，利用细腻的振动反馈提升交互质感。

---

## 2. 25页详细功能与界面设计

### 模块一：引导与身份验证 (Onboarding & Auth)

#### Page 1: 启动页 (Splash Screen)
*   **UI 元素**：全屏纯色背景（#F9F9F9），居中一个极简圆圈 Logo，底部一行微弱的浅灰色版本号。
*   **交互与动画**：Logo 伴随淡入（Fade-in）和轻微放大，1.2 秒后平滑平移至 Page 2。

#### Page 2: 欢迎与登录选择页 (Welcome Screen)
*   **UI 元素**：
    *   大号标题（H1）：“你好。\n开始你的健康旅程。”
    *   下方三个无边界扁平按钮：“手机号一键登录”、“微信登录”、“暂不登录，先体验”。
*   **交互**：点击手机号登录后，整体页面向左滑出，Page 3 滑入。

#### Page 3: 验证码输入页 (OTP Screen)
*   **UI 元素**：
    *   提示语：“我们向您的手机发送了验证码”。
    *   4 个独立的、只有底线的数字输入框。
    *   调用系统纯数字软键盘。
*   **交互**：每输完一个数字，光标自动跳到下一格。输满 4 位自动触发网络请求。

#### Page 4: 权限授权引导页 (Permissions Screen)
*   **UI 元素**：
    *   垂直排列的两个线性图标：步数读取权限（通知、身体传感器）。
    *   极简文案：“为了精准记录您的每一步。”
    *   底部大按钮：“授权并开启”。

### 模块二：首次健康档案初始化 (Health Profiling)
*注：采用一页一问的向导模式（Wizard Flow）。*

#### Page 5: 性别选择页 (Gender Selection)
*   **UI 元素**：
    *   标题：“你的性别是？”
    *   屏幕中央两个大号线性圆形图标（男 / 女）。
*   **交互**：点击其中一个后，该圆圈背景变为低饱和度强调色，并在 0.3 秒后自动切页。

#### Page 6: 出生年月页 (Age Selection)
*   **UI 元素**：
    *   标题：“你的出生年份？”
    *   屏幕中央一个半透明的垂直滚轮选择器（Wheel Picker）。
*   **交互**：滚动时带有轻微的齿轮机械震动反馈。

#### Page 7: 身高录入页 (Height Input)
*   **UI 元素**：
    *   顶部巨大数字：“180 cm”。
    *   屏幕下方一个垂直的滑动刻度尺。
*   **交互**：手指上下滑动刻度尺，顶部的数字实时平滑放大或缩小变化。

#### Page 8: 体重录入页 (Weight Input)
*   **UI 元素**：
    *   顶部巨大数字：“75.0 kg”。
    *   屏幕下方一个水平方向的滑动刻度尺。
*   **交互**：左右滑动调整体重，刻度每移动 0.1kg 伴随一次微弱的震动。

#### Page 9: 目标设定页 (Goal Setting)
*   **UI 元素**：
    *   标题：“你的主要目标是？”
    *   三个纵向排列的纯文本大卡片：“减脂”、“增肌”、“保持健康健康”。
*   **交互**：卡片无边框，选中时背景色变成非常淡的强调色。

#### Page 10: 档案定制完成页 (Setup Complete)
*   **UI 元素**：
    *   屏幕中央一个缓缓画圆的打钩动画（Vector Animation）。
    *   计算结果文本：“已为您激活基础代谢：1750 kcal/日”。
    *   底部按钮：“开启极简健康”。

---

### 模块三：核心主导航区 (Main Tabs)

#### Page 11: 首页 - 今日看板 (Dashboard)
*   **UI 元素**：
    *   顶部核心区：大字展示当前“热量缺口（-400 kcal）”，外加三个同心极简圆环（摄入、消耗、步数进度）。
    *   中部：三个列表项（饮食记录、运动记录、饮水进度条）。
    *   右下角：一个纯圆形的悬浮操作按钮（FAB），仅带一个 “+” 号。
*   **交互**：点击 FAB 展开一个全屏半透明遮罩，浮现“记饮食”、“记运动”、“记饮水”三个文字图标。

#### Page 12: 日志页 - 时间轴 (Timeline Diary)
*   **UI 元素**：
    *   左侧一条贯穿全屏的细灰线（时间轴）。
    *   右侧按时间点排列的极简文字流（例如：*08:30 早餐 450 kcal*，*19:00 户外跑步 10.03km*）。
*   **交互**：向下滑动可无限加载历史日子；长按某条记录可弹出“删除”选项。

#### Page 13: 计划与习惯页 (Habit Plans)
*   **UI 元素**：
    *   顶部“正在进行的计划”卡片（如：16:8 断食计划，显示当前断食倒计时）。
    *   下方每日习惯打卡清单（如：今日冥想、今日拉伸），右侧带一个极简的复选框。

#### Page 14: 我的页 (Profile Settings)
*   **UI 元素**：
    *   顶部用户头像（极简纯色圆形内含姓氏首字母）与名字。
    *   下方一列纯文字菜单：“身体指标历史”、“我的勋章墙”、“数据导出 (CSV)”、“系统设置”。

---

### 模块四：饮食与水追踪 (Diet & Hydration)

#### Page 15: 添加饮食搜索页 (Search Food)
*   **UI 元素**：
    *   顶部全宽无边框搜索框，带有一个闪烁的光标。
    *   下方展示“常吃食物”与“最近搜索”的标签流（Tag Cloud）。

#### Page 16: 食物详情与份量页 (Food Detail & Portion)
*   **UI 元素**：
    *   顶部食物线性图标及每百克热量。
    *   中部：碳水、蛋白质、脂肪三条水平对比柱状图。
    *   底部：一个圆形的克数调节盘（如 100g, 200g）。

#### Page 17: 扫码/Camera 智能识别页 (Camera Scan)
*   **UI 元素**：
    *   全屏相机取景框。
    *   中央一个白色细线组成的扫描方框。
    *   底部一个极简的“拍照识别”圆形快门键。

#### Page 18: 饮水记录页 (Water Log)
*   **UI 元素**：
    *   全屏淡蓝色背景，画面下半部分为动态的水波纹效果（Wave Animation）。
    *   屏幕正中央显示大号字“今日已饮水: 1250 ml / 2000 ml”。
*   **交互**：轻击屏幕任意位置，水波纹向上平滑溢满 250ml；长按屏幕任意位置撤销上一次记录。

---

### 模块五：运动追踪 (Activity Tracker)

#### Page 19: 运动类型选择页 (Select Workout)
*   **UI 元素**：
    *   九宫格/网格布局，展示多种运动类型（户外跑步、室内跑步、骑行、步行、力量训练）。
    *   每个格子仅由一个简单的线性图标和文字组成。

#### Page 20: 运动进行中全屏页 (Active Session)
*   **UI 元素**：
    *   全屏纯黑沉浸式背景（AMOLED 节能且防误触）。
    *   上方巨大白色数字：时长（01:04:23）。
    *   下方较小数字：当前心率（145 bpm）、已消耗热量（719 kcal）、当前配速（6'24"）。
*   **交互**：底部一个长条状的“滑动结束”控件，用户必须从左滑到右才能停止运动，杜绝汗水误触。

#### Page 21: 运动总结报告页 (Workout Summary)
*   **UI 元素**：
    *   上半部分：单色极简地图线条（如果是户外 GPS 运动），标出起终点。
    *   下半部分：各项核心指标卡片，配以极简的心率区间直方图。

---

### 模块六：健康数据深度分析与设置 (Analytics & Meta)

#### Page 22: 体重趋势分析页 (Weight Trend Chart)
*   **UI 元素**：
    *   全屏宽度折线图。没有任何网格线和复杂的坐标轴，只有一条平滑的渐变色曲线。
    *   点击曲线上任意一点，上方悬浮浮现当日的具体体重数值。

#### Page 23: 热量收支周报页 (Weekly Energy Report)
*   **UI 元素**：
    *   7 组垂直的双向柱状图。上方柱子代表摄入，下方柱子代表运动消耗。
    *   用一条水平虚线代表用户设定的目标基础代谢线。

#### Page 24: 睡眠手动录入页 (Sleep Log Input)
*   **UI 元素**：
    *   屏幕中央是一个 24 小时的圆环时钟表盘。
    *   表盘上有两个可滑动的月亮和太阳滑块，分别代表“入睡时间”与“醒来时间”。
*   **交互**：拖动月亮和太阳滑块来直观框选睡眠区间。

#### Page 25: 身体围度记录页 (Body Measurements)
*   **UI 元素**：
    *   左侧是一个极简的人体线条侧面剪影框图。
    *   右侧是对应剪影位置的输入框列表：胸围、腰围、臀围、大腿围。
    *   点击右侧输入框时，左侧剪影对应的身体部位会自动高亮闪烁。

## 3. 技术选型与实现架构建议

### 3.1 UI 技术栈
*   **声明式 UI 框架**：**Jetpack Compose** (全面取代传统 XML 布局)。
*   **单 Activity 架构**：全局仅用一个 `MainActivity`，依托 `navigation-compose` 管理上述 25 个 Composable 视图，确保页面转场动画具有 120Hz 的绝对丝滑度。
